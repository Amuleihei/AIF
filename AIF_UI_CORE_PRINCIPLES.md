# AIF UI Core Principles

## Position

AIF 的核心产品要求，不只有业务流程与三语环境。

从现在开始，`核心 UI 风格 / 交互手感` 与 `中文 / English / မြန်မာ 三语环境` 视为同一级别的系统资产，属于 AIF 的长期核心能力，不是可有可无的前端装饰。

## Core Rule

以后新增或改动 Web / H5 / 关键弹层时，默认遵守以下原则：

1. 要有“轻、快、稳、顺手”的系统级交互感。
2. 优先使用就地操作、浮层、气泡、局部反馈，减少整页跳转和来回滚动。
3. 交互应尽量接近 macOS / iPhone 那种自然反馈：
   - 轻微缩放
   - 柔和阴影
   - 半透明 / 玻璃感
   - 按压反馈
   - 出现与消失都有过渡
4. 界面必须服务生产现场，不能为了“炫”牺牲效率。
5. 视觉统一性优先于局部随意发挥。
6. 三语界面切换时，UI 结构、层级、操作感必须保持一致。

## AIF Preferred Interaction Style

适合 AIF 的默认交互方向：

- 卡片点击后原地展开操作，不强依赖顶部工具栏
- 重要操作优先用气泡菜单、小浮层、底部弹层
- 列表页避免让用户上下滚动寻找动作
- 管理页优先“直接操作”而不是“先选中再去别处点按钮”
- 状态反馈尽量即时、局部、明确

## Forbidden Direction

以下方向默认不推荐：

- 纯后台工具感、只有表格没有交互层次
- 每个动作都跳新页面
- 操作必须先选人、再滚到顶部、再找按钮
- 出现/关闭没有过渡，像旧式管理系统
- 只做“能用”，不管手感和效率

## Priority Level

在 AIF 项目里，以下三项属于长期固定优先级：

1. 业务流程真实可用
2. 三语环境一致可用
3. 核心 UI 风格与操作手感一致可用

## Implementation Note

如果未来出现“功能能做完，但会破坏 AIF 当前核心 UI 风格”的情况，应该优先停下来重构交互，而不是直接并入一个生硬方案。
